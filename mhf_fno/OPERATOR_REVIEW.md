## 🔬 深度分析

### 1. TFNO 为什么不适合 MHF 替换？

**TFNO 的核心**：张量化（Tensorization）

```
标准 SpectralConv:
  weight: (in_channels, out_channels, n_modes)
  
TFNO SpectralConv:
  weight: 分解为低秩张量
  例如: (in, out, modes) → (in, r1) × (r1, out, modes/r1)
  
参数减少: in × out × modes → in × r1 + r1 × out × (modes/r1)
```

**问题**:
- TFNO 已经使用张量化压缩参数 (24,802 vs FNO 133,873)
- MHF 替换后参数增加 (24,802 → 36,076, +45%)
- 两层压缩机制冲突

**结论**: TFNO 不需要 MHF，已有更好的压缩方式

---

### 2. UNO 的多尺度问题

**UNO 的核心**：U-Net 风格的多尺度结构

```
编码器 (下采样):
  Block 0: 32 → 32 (高分辨率)
  Block 1: 32 → 64 (下采样)

瓶颈:
  Block 2: 64 → 64 (最低分辨率)

解码器 (上采样):
  Block 3: 128 → 64 (上采样 + skip connection)
  Block 4: 96 → 32 (高分辨率)

通道数不规则: 128, 96 (因为 skip connection)
```

**问题**:
1. **n_modes 不匹配**: UNO 使用 [8, 5] (rfft 后)，MHF 设计假设 [8, 8]
2. **通道数不规则**: 128, 96 不是标准倍数，MHF 分头困难
3. **skip connection**: 改变 SpectralConv 会影响 skip connection

**MHF-PyramidUNO 失败原因**:
- 参数减少 78% (812,193 → 175,457)
- 但误差很高 (0.5687 vs 0.1068)
- 原因: 简化版丢失了 UNO 的核心结构 (skip connection)

---

### 3. FNO 边缘层 MHF 为什么有效？

**关键洞察**:

```
FNO 结构:
  Layer 0: 输入特征提取 (全局信息)
  Layer 1: 特征变换
  Layer 2: 输出整合 (生成结果)

边缘层 (0, 2):
  - Layer 0: 捕获输入的全局趋势
  - Layer 2: 整合输出特征
  
中间层 (1):
  - 需要保留完整频域信息
  - MHF 分头会丢失细节
```

**验证数据**:

| 配置 | 参数量 | L2误差 | 分析 |
|------|--------|--------|------|
| 全部 MHF | 41,713 | 0.1319 (+37.9%) | 过度压缩 |
| 边缘 MHF | 72,433 | 0.0966 (+1.0%) | 平衡点 ✅ |

## 💡 新设计原则

### 原则 1: 不是所有算子都适合 MHF

```
适用 MHF:
  - FNO: 简单结构，边缘层有效
  - 自定义算子: 可以从零设计 MHF

不适用 MHF:
  - TFNO: 已有张量化压缩
  - UNO: 多尺度结构复杂
  - SFNO: 球面谐波有物理意义
```

### 原则 2: MHF 的真正价值是隐式正则化

```
场景分析:

小数据集 (<1000 样本):
  - MHF 有效: 防止过拟合
  - 参数少 → 泛化好

大数据集 (>10000 样本):
  - MHF 可能限制表达能力
  - 不需要正则化

UNO 失败原因:
  - 参数减少 78% → 表达能力不足
  - UNO 本身参数多，需要完整表达能力
```

### 原则 3: 新算子设计需要尊重原有架构

```
MHF-PyramidUNO 失败:
  - 去掉了 skip connection
  - 破坏了 UNO 的核心机制

正确做法:
  - 保留 UNO 的 skip connection
  - 仅在卷积层内部优化
```

## 📝 结论

1. **FNO + MHF 边缘层** 是唯一验证有效的组合
2. **TFNO 不需要 MHF**，已有更好的张量化压缩
3. **UNO 需要保留核心结构**，不能简单替换
4. **新设计应该围绕小数据集正则化**，而不是盲目减少参数
