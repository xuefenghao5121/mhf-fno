## 测试 1: FNO 深度验证 ✅

### 1.1 配置变化测试

**目的**: 确定最佳 MHF 配置

**测试矩阵**:

| 配置 | mhf_layers | n_heads | 参数量预期 |
|------|------------|---------|-----------|
| 基准 | [] | 1 | 133,873 |
| 全MHF-2头 | [0,1,2] | 2 | ~67K |
| 全MHF-4头 | [0,1,2] | 4 | ~42K |
| 边缘-2头 | [0,2] | 2 | ~103K |
| 边缘-4头 | [0,2] | 4 | ~72K |
| 边缘-8头 | [0,2] | 8 | ~42K |

**评估指标**:
- L2 误差
- 训练稳定性 (标准差)
- 收敛速度

### 1.2 hidden_channels 影响

**目的**: 验证 MHF 在不同模型大小下的效果

**测试配置**:

| hidden | FNO 参数 | MHF 参数 | 测试 |
|--------|----------|----------|------|
| 16 | ~34K | ~18K | 小模型 |
| 32 | ~134K | ~72K | 标准模型 |
| 64 | ~534K | ~289K | 大模型 |
| 128 | ~2.1M | ~1.1M | 超大模型 |

**假设**: hidden 越大，MHF 效果越好（参数效率更重要）

### 1.3 数据量影响

**目的**: 验证 MHF 的正则化效果

**测试配置**:

| 样本数 | 预期 MHF 优势 |
|--------|---------------|
| 100 | 高（过拟合风险大）|
| 250 | 中高 |
| 500 | 中 |
| 1000 | 低 |

**假设**: 数据量越小，MHF 效果越好

## 测试 2: TFNO 验证 ⏳

### 2.1 TFNO vs TFNO+MHF

**目的**: 验证叠加效果

**测试配置**:

```python
# 标准 TFNO
from neuralop.models import TFNO
model_tfnno = TFNO(n_modes=(8,8), hidden_channels=32)

# TFNO + MHF (仅边缘层)
# 需要检查 TFNO 是否支持 conv_module 参数
```

**假设**: 叠加效果为负（过度压缩）

**验证方法**:
1. 对比参数量
2. 对比 L2 误差
3. 如果 TFNO+MHF > TFNO，说明过度压缩

### 2.2 补偿策略测试

**目的**: 如果叠加为负，测试补偿策略

**补偿方法**:
1. 减少 n_heads (4→2)
2. 增加 hidden_channels (32→48)
3. 增加 n_modes ((8,8)→(12,12))

## 测试 3: UNO 验证 ⏳

### 3.1 位置选择测试

**目的**: 确定 MHF 在 UNO 的最佳位置

**UNO 结构**:
```
编码器 (下采样): high_res → mid_res → low_res
瓶颈: bottleneck
解码器 (上采样): low_res → mid_res → high_res
```

**测试矩阵**:

| 位置 | 假设效果 |
|------|----------|
| 编码器 (高分辨率) | ❌ 可能丢失细节 |
| 瓶颈 | ✅ 参数效率 |
| 解码器 (低分辨率) | ✅ 隐式正则化 |
| 边缘层 (输入+输出) | ✅ 推荐 |

### 3.2 UNO + MHF 实现

```python
from neuralop.models import UNO

# 检查 UNO 是否支持 conv_module
model = UNO(
    in_channels=1,
    out_channels=1,
    hidden_channels=32,
    # ...
)
```
